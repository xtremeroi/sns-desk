# S&S Desk

Menu bar companion for [Stats & Strats](https://sns.xtremeroi.com): the punch
clock in your Mac's menu bar, plus desktop-wide time tracking that the browser
can't see.

**This code is public on purpose.** S&S Desk observes activity on your
computer while you're clocked in, so everyone on the team can read exactly
what it does and doesn't collect. The short version, verifiable in
[lib/tracker.js](lib/tracker.js):

- It records the **name** of the frontmost application (e.g. "Chrome",
  "Excel") — never window titles, URLs, keystrokes, screenshots, or content.
  It never records itself.
- The tracker runs **only while you are clocked in**. Off the clock, nothing
  is observed or recorded at all — not locally, not on the server. The punch
  clock is the on/off switch for all tracking, and you can exclude any app
  entirely (⊘ button).
- Quitting the app (footer button or right-click the menu bar icon → Quit)
  stops all tracking on that machine immediately.

## What it does

- **Punch clock in the tray** — live `h:mm:ss` ticker while clocked in, with
  clock in / break / resume / clock out and client attribution. Speaks the
  exact same `/api/store` punch protocol as the web app, so the header pill,
  Time page, and billing views all stay in sync.
- **Desktop-wide activity tracking, only while clocked in** — samples the
  frontmost application every 30s (app **names only**, never window titles or
  content; idle-gated at 5 min). Clocked out = the tracker is off entirely.
  Recorded blocks show in the popup's "Today on this Mac" view and post to
  the S&S timesheet (`timeseg`, `c: "app:<name>"`). Per-app opt-out via the
  ⊘ button on any row.
- **Offline queue** — failed punches and segments persist to disk with real
  timestamps and replay in order on reconnect (server accepts `at` ≤36h back),
  same contract as the web app's `punch-pending-q`.
- **Lock/sleep safety net** — away longer than the manager-set grace
  (`punchLockMin`, default 45 min) while clocked in → retroactive clock-out at
  lock time, flagged auto (`outAt` + `lock`, same as the Chrome extension) with
  a notification.

## Auth

No credentials are stored. A login window completes the normal Auth0 +
Cloudflare Access flow at sns.xtremeroi.com once; the persisted
`CF_Authorization` cookie (24h session) authenticates API calls. When it
expires, the tray shows ⚠ and the popup offers "Sign in" — tracked time queues
locally in the meantime.

## Quit vs. close

- The **✕** in the popup just closes the panel — the app keeps running (and
  tracking) in the menu bar, like any menu bar utility.
- To actually stop: **"Quit S&S Desk — stop tracking"** in the popup footer,
  or right-click the menu bar icon → Quit. If you're still clocked in, it
  asks whether to clock you out first (the punch clock lives on the server
  and would otherwise keep running).

## Run from source

```
npm install
npm start
```

Look for **S&S** in the menu bar. Click it for the popup.

## Package

```
npm run dist   # → dist/S&S Desk-<version>-universal.dmg (unsigned)
```

Icon is generated by `python3 scripts/make_icon.py` (requires Pillow) and
committed at `build/icon.icns`.

## Data locations

`~/Library/Application Support/sns-desk/`
- `days/YYYY-MM-DD.json` — local activity blocks (personal view)
- `pending.json` / `punch-pending.json` — offline queues
- `settings.json` — excluded apps
